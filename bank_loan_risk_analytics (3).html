Dataset & Schema
About the Dataset

This project uses the public Loan Default Prediction Dataset from Kaggle — a real-world credit analytics dataset with ~10,000 loan records.

Fields include borrower demographics, financial indicators, and loan outcome status — giving a full picture of credit risk at the individual level.

income credit_score loan_amount dti_ratio loan_purpose employment_length loan_status
Table Schema
CREATE TABLE loans ( borrower_id INT, credit_score INT, income DECIMAL(10,2), loan_amount DECIMAL(10,2), loan_purpose VARCHAR(50), dti_ratio DECIMAL(4,2), employment_length INT, loan_status VARCHAR(20) );

SQL Analysis — Risk Classification
bank_loan_risk.sql — Mohit Das
-- ============================================================ -- Bank Loan Risk Analytics — Mohit Das -- Default rate segmentation by credit score band + DTI -- ============================================================ -- STEP 1: Data Cleaning SELECT * FROM loans WHERE credit_score BETWEEN 300 AND 850 AND dti_ratio BETWEEN 0 AND 1 AND loan_amount > 0 AND income > 0; -- STEP 2: Default Rate by Credit Score Band SELECT CASE WHEN credit_score < 580 THEN 'Poor (<580)' WHEN credit_score BETWEEN 580 AND 669 THEN 'Fair (580–669)' WHEN credit_score BETWEEN 670 AND 739 THEN 'Good (670–739)' ELSE 'Excellent (740+)' END AS score_band, COUNT(*) AS total_borrowers, SUM(CASE WHEN loan_status = 'Default' THEN 1 ELSE 0 END) AS defaults, ROUND(100.0 * SUM(CASE WHEN loan_status = 'Default' THEN 1 ELSE 0 END) / COUNT(*), 1) AS default_rate_pct FROM loans GROUP BY score_band ORDER BY default_rate_pct DESC; -- STEP 3: Exposure by Loan Purpose SELECT loan_purpose, COUNT(*) AS loan_count, ROUND(SUM(loan_amount) / 10000000, 1) AS exposure_cr, ROUND(AVG(dti_ratio) * 100, 1) AS avg_dti_pct, ROUND(100.0 * SUM(CASE WHEN loan_status = 'Default' THEN 1 ELSE 0 END) / COUNT(*), 1) AS default_rate_pct FROM loans GROUP BY loan_purpose ORDER BY exposure_cr DESC; -- STEP 4: High-Risk Borrower Watchlist SELECT borrower_id, credit_score, dti_ratio, loan_amount, loan_purpose, loan_status, ROUND( (1 - (credit_score - 300) / 550.0) * 40 + dti_ratio * 35 , 1) AS risk_score FROM loans WHERE credit_score < 580 OR dti_ratio > 0.50 ORDER BY risk_score DESC LIMIT 20; -- STEP 5: Portfolio Summary with Window Functions SELECT score_band, total_borrowers, defaults, default_rate_pct, ROUND(100.0 * total_borrowers / SUM(total_borrowers) OVER(), 1) AS pct_of_portfolio, SUM(defaults) OVER (ORDER BY default_rate_pct DESC) AS cumulative_defaults FROM score_band_summary ORDER BY default_rate_pct DESC;

Key Insights & Recommendations
Key Findings
Borrowers with credit score below 580 have the highest default rate (~38%).
Higher DTI ratio is associated with greater default risk.
Personal loans show higher risk than other loan types.
Most borrowers fall in the Fair band (580–669) — that's where the bulk of portfolio risk sits.
Higher income borrowers tend to default less, regardless of credit score.
Recommendations
Be stricter with DTI above 45% — maybe ask for a co-signer or collateral.
Charge higher interest rates for Poor/Fair score borrowers to cover the risk.
Keep a closer eye on personal loans — they seem to go wrong more often.
Use SQL flags to catch risky borrowers early, before they actually default.
Try to grow the Excellent score segment — lower risk, more predictable repayments.
Limitations & Honest Caveats
⚠ Things I'd flag before anyone acts on this
· Dataset is a simulated Kaggle credit dataset — not real bank data
· Does not include repayment history or missed payment counts
· Macroeconomic factors (inflation, interest rates) are not modelled
· Risk score is a simplified formula — not a validated credit model
· No time-series data — cannot track borrower behaviour over time
· Default labels are rule-based in this sample, not observed outcomes

A note from me
This was my first end-to-end SQL analytics project. I built it to practice writing multi-step queries — cleaning, aggregating, and using window functions on a realistic dataset.
