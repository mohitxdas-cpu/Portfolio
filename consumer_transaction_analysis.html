Project Overview
What this project explores

This project analyzes a simulated retail transaction dataset to surface patterns in how customers spend — segmented by region, purchase frequency, and behavioral signals like recency and basket size.

The goal was to practice interpreting usage patterns the same way an analyst would at a company like GlobalLogic: identify where demand concentrates, flag customers showing churn signals, and communicate findings in a format management can act on.

The output is this dashboard, a structured summary report, and a set of actionable observations for a simulated operations audience.

Analytical questions I set out to answer
Which regions drive the most transaction volume?
Do high-frequency customers also have the highest spend?
What behavioral signals predict churn before it happens?
Which customer segments are most valuable to retain?
How does purchase frequency vary by geography?
Where are the anomalies in transaction trends?

Key SQL Logic — RFM Segmentation Query
customer_rfm_segments.sql
-- Step 1: Calculate Recency, Frequency, Monetary value per customer WITH rfm_base AS ( SELECT customer_id, region, DATEDIFF(CURDATE(), MAX(transaction_date)) AS recency_days, COUNT(transaction_id) AS frequency, SUM(transaction_amount) AS monetary_value FROM transactions WHERE transaction_date >= DATE_SUB(CURDATE(), INTERVAL 180 DAY) GROUP BY customer_id, region ), -- Step 2: Score each dimension 1–3 rfm_scored AS ( SELECT *, CASE WHEN recency_days <= 30 THEN 3 WHEN recency_days <= 60 THEN 2 ELSE 1 END AS r_score, CASE WHEN frequency >= 10 THEN 3 WHEN frequency >= 5 THEN 2 ELSE 1 END AS f_score, CASE WHEN monetary_value >= 50000 THEN 3 WHEN monetary_value >= 20000 THEN 2 ELSE 1 END AS m_score FROM rfm_base ) -- Step 3: Classify into segments and flag churn risk SELECT customer_id, region, recency_days, frequency, ROUND(monetary_value, 2) AS total_spend, (r_score + f_score + m_score) AS rfm_score, CASE WHEN (r_score + f_score + m_score) >= 8 THEN 'High-Value' WHEN r_score = 1 AND f_score >= 2 THEN 'At-Risk' WHEN recency_days > 60 AND frequency < 3 THEN 'Churn' ELSE 'Active' END AS segment FROM rfm_scored ORDER BY rfm_score DESC;

Key Findings & Operational Recommendations
📍 Geographic Concentration
North and West regions account for nearly half of total transaction volume despite having only ~35% of the customer base. This suggests higher per-customer engagement driven by income levels and urban retail density. Operations resources should be weighted toward these regions for maximum impact.
⚠️ At-Risk Segment Alert
A notable share of previously high-frequency customers have not transacted in 31–60 days — the classic "slipping away" window. These customers have high RFM scores historically, which means they are worth re-engaging. A targeted outreach within this window typically has a higher recovery rate than waiting for full churn.
💎 High-Value Retention Priority
High-Value customers (RFM ≥ 8) represent a disproportionate share of total spend relative to their count. Losing even a small number from this segment has an outsized negative impact on revenue. Operational quality, response time, and relationship management should be prioritized for this group above all others.

A note from me
I built this project to practice the kind of analysis I'd expect to do in an operations or analytics role — identifying patterns in transaction data, segmenting customers by behavior, and communicating findings clearly. The RFM framework came from reading about how e-commerce and banking teams actually classify customers; I simplified it to three buckets to keep the logic transparent and explainable.

The geographic angle was something I added after noticing that a lot of real-world operations decisions (staffing, outreach, quality targets) are geography-driven. If the numbers look clean, that's intentional — I seeded the data carefully so the patterns are realistic. If you have feedback, I'd genuinely like to hear it.
